# 2026-03-13 - 技能创建经验总结

## 重要学习：如何正确创建OpenClaw技能

今天通过创建"中医方剂卡片制作技能"，深入学习了skill-creator的正确使用方法，总结了宝贵的经验教训。

### 核心原则

1. **技能的本质**：技能是给大模型使用的指导文档，不是给人类用户的操作手册
2. **渐进式披露**：三级加载系统（元数据 → SKILL.md概述 → 详细资源）
3. **为大模型设计**：考虑AI如何使用，而不是人类如何使用

### 标准目录结构
```
skill-name/
├── SKILL.md (必需)           # 工作流程概述
├── references/ (可选)        # 详细规范、规则、示例
├── scripts/ (可选)          # 可执行脚本
├── assets/ (可选)           # 输出模板、资源文件
└── .env (可选)              # 环境配置
```

### 关键经验教训

#### 1. SKILL.md要保持简洁
- **错误做法**：将所有内容堆在SKILL.md中
- **正确做法**：SKILL.md只包含工作流程概述和文件引用
- **标准**：少于500行，少于5000字

#### 2. 正确的文件分类
- **模板文件** → `assets/`（不是`templates/`）
- **参考文档** → `references/`
- **可执行代码** → `scripts/`
- **环境配置** → `.env`

#### 3. 避免常见陷阱
- **循环引用**：文件相互引用 → 改为线性引用
- **重复内容**：相同信息多处出现 → 只在一处定义
- **缺乏指引**：大模型不知道读哪个文件 → 明确指示

#### 4. 渐进式披露设计
1. **SKILL.md**：触发后加载，包含工作流程概述
2. **references/**：按需加载，包含详细规范
3. **scripts/assets**：执行时使用，不加载到上下文

### 从中医方剂卡片技能学到的具体教训

1. **初始错误**：创建了`templates/`目录
   - **修正**：改为`assets/`目录
   - **原因**：skill-creator标准中只有scripts/, references/, assets/

2. **初始错误**：SKILL.md包含太多细节
   - **修正**：只保留概述，细节移到references/
   - **原因**：保持上下文简洁

3. **初始错误**：文件之间有循环引用
   - **修正**：设计线性工作流程
   - **原因**：避免引用混乱

4. **初始错误**：缺乏明确指引
   - **修正**：在SKILL.md中明确指示何时读取哪个文件
   - **原因**：大模型需要明确指导

### 最佳实践

1. **文件命名**：使用kebab-case，名称明确反映内容
2. **引用格式**：`[文件名](路径)`，说明何时读取
3. **内容组织**：每个文件一个主题，提供完整示例
4. **模板设计**：参数化，有默认值，包含示例
5. **脚本编写**：有错误处理，支持命令行参数
