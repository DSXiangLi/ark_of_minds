# MEMORY.md - 长期记忆

## 技能创建专业知识

### 核心原则
1. **技能本质**：给大模型使用的指导文档，不是人类操作手册
2. **渐进式披露**：三级加载（元数据 → SKILL.md概述 → 详细资源）
3. **为大模型设计**：考虑AI如何使用，提供明确指引

### 标准目录结构
```
skill-name/
├── SKILL.md          # 工作流程概述（<500行）
├── references/       # 详细规范、规则、示例
├── scripts/         # 可执行脚本
├── assets/          # 输出模板、资源文件
└── .env             # 环境配置
```

### 关键经验教训
1. **SKILL.md要简洁**：只包含概述和文件引用，细节在references/
2. **正确文件分类**：模板在assets/，不在templates/
3. **避免循环引用**：设计线性工作流程
4. **明确引用指引**：在SKILL.md中说明何时读取哪个文件

### 常见错误及修正
1. **错误**：SKILL.md太详细 → **修正**：细节移到references/
2. **错误**：创建templates/目录 → **修正**：使用assets/目录
3. **错误**：文件循环引用 → **修正**：设计线性引用
4. **错误**：缺乏使用指引 → **修正**：明确指示文件读取时机

### 最佳实践
1. 文件命名：kebab-case，名称明确
2. 引用格式：[文件名](路径)，说明何时读取
3. 内容组织：每个文件一个主题，提供完整示例
4. 模板设计：参数化，有默认值
5. 脚本编写：有错误处理，支持命令行
