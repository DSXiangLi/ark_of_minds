# DeepSeek 问答爬虫项目 - 核心经验总结

> 从实际项目中提炼的第一性原理、可复用技巧与最佳实践

---

## 一、问题的本质

```
用户需求：自动化获取 DeepSeek AI 的回答 + 引用链接 → 保存为 JSON

表面问题：提取失败、超时循环、引用丢失
本质问题：多模态 Agent 与动态网页的同步问题
```

**核心矛盾**：
- Agent 的 `browser_state` 是**快照**，但网页是**动态渲染**
- AI 生成回答是**流式输出**，但提取是**瞬时操作**
- 历史对话和当前对话在 DOM 中**混合共存**

---

## 二、关键技术决策与迭代

### 1. 智能等待 vs 固定等待

| 方案 | 结果 | 原因 |
|------|------|------|
| 固定等待30秒 | 部分失败 | AI生成时间不可预测 |
| JS轮询检测 | 代码复杂 | 等待逻辑混入提取工具 |
| 简化任务指令 | ✅ 成功 | 减少 Agent 决策复杂度 |

**第一性原理**：与其增强工具复杂度，不如简化 Agent 的决策空间

### 2. 答案提取策略演进

```
原始方案：查找包含"已阅读"的元素
    ↓
问题：侧边栏历史也包含"已阅读"
    ↓
优化：lastIndexOf 找最后一个
    ↓
问题：非搜索模式（只有"已思考"）
    ↓
最终：多标志检测 + 多重清理
```

**核心学习**：不要假设单一方案覆盖所有场景，要设计**降级策略链**

### 3. 引用提取的陷阱

```javascript
// 错误：严格过滤
if (!isKnown) return;  // 丢失大量有效链接

// 正确：宽松提取 + 去重
if (text.length >= 2 || href.length > 20)  // 接受更多
```

**教训**：数据提取时，**保真 > 过滤**，后期可以去重

## 三、调试方法论的突破

### 传统方法的局限

```
console.log → 只看到结果，看不到过程
print → 无法看到 Agent 内部决策
```

### 解决方案：全链路日志

```python
# 核心代码
save_conversation_path=str(debug_dir / "conversation.json")
```

记录了：
- 每步的 `thinking`（模型推理）
- 每步的 `browser_state`（页面快照）
- 每步的截图

### 关键洞察

从日志发现：
```
Step 3: Agent 收到"发送消息后"的browser_state
Step 4: 但页面实际已显示AI回复
    ↓
原因：browser_state 是上一次轮询的快照，不是实时状态
```

## 四、系统设计的核心原则

### 1. 最小化 Agent 决策空间

```python
# 复杂指令（容易迷失）
"1. 打开网页 2. 输入问题 3. 启用深度思考 4. 启用智能搜索 ..."

# 简化指令（稳定）
"1. 输入问题发送 2. 等待回复 3. 保存结果"
```

**原理**：Agent 能力 = 理解指令 × 决策正确率  
减少决策点 = 提高成功率

### 2. 提取工具的容错设计

```python
# 三层降级
try:
    # 方法1: 优先找"已阅读"
except:
    try:
        # 方法2: 找"已思考"  
    except:
        # 方法3: 直接取body
```

**原理**：错误成本从低到高排列，优先尝试最高效方案

### 3. 调试信息的分层记录

| 层级 | 内容 | 用途 |
|------|------|------|
| 控制台 | 关键指标 | 快速判断 |
| JSON | 完整回答+引用 | 结果验证 |
| conversation | Agent推理过程 | 定位根因 |
| 截图 | 页面快照 | 可视化验证 |

## 五、可复用原则清单

### 设计原则

```
1. 设计：最小可行 → 快速迭代
2. 调试：全链路日志 → 差异分析
3. 提取：降级策略链 → 宽松+去重
4. Agent：简化指令 → 明确完成标志
5. 错误：分层处理 → 兜底方案
6. 测试：边界覆盖 → 回归验证
7. 迭代：观察→假设→验证
8. 心态：简化 > 复杂，日志 > 猜测
```

### 技术要点

- **动态内容检测**：找到"完成标志"后再提取
- **字符串清洗 > DOM选择**：内容特征比结构稳定
- **宽松提取 + 严格去重**：先保数据，后优化
- **日志优于猜测**：永远基于证据做决策

## 六、核心反思

> 技术的本质不是解决问题，而是管理复杂性。

当我们无法完全控制一个系统时：
1. 记录它的行为
2. 理解它的模式
3. 在它的框架内工作
4. 用约束替代控制

这就是与 AI Agent 系统工作的第一性原理。
